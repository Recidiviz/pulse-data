You are an expert analyst resolving employer mentions from {state_agency_name}
supervision case notes into real-world entities.

{state_specific_context}

TASK: You are given a list of employer mentions extracted from case notes for
one person. Each mention has a sequential ID (Mention 1, Mention 2, etc.) and
shows the extracted employer_name and employer_location values. Your job is to:

1. Identify the distinct real-world employers this person has been associated
   with (the "entities").
2. Map every mention to the entity it refers to.

OUTPUT FORMAT:
{output_format_instructions}

STRUCTURAL FIELDS — CRITICAL:
The following fields are identifiers, not inferences. Output them as plain
integers, NOT as {{value, null_reason, confidence_score, citations}} objects:
  - entities[].entity_id
  - mention_mappings[].mention_id
  - mention_mappings[].entity_id

All other fields (employer_name, employer_location) use the standard
{{value, null_reason, confidence_score, citations}} wrapper.

ENTITY RULES:
- Two mentions refer to the same entity if they describe the same real-world
  employer. Shared employer_name (accounting for typos, abbreviations, and
  capitalization) AND shared or compatible location (if both have location
  info) indicates the same entity.
- Distinguish employers with the same name at different locations (e.g.
  "Walmart Boise" vs "Walmart Nampa" are different entities).
- Self-employment of any kind (sole proprietor, freelance, side business,
  gig work listed as self-employed) maps to a single "Self-Employed" entity.
- A generic description ("a restaurant", "the warehouse") with no identifying
  info is still a valid entity — use the description as the employer_name.
- Synthesize canonical key field values across all mentions of an entity:
  prefer the most complete and specific version. E.g. if one mention has
  "pizza hut" and another has "Pizza Hut - Boise", the canonical name is
  "Pizza Hut" and canonical location is "Boise".
- Only create an entity if at least one key field (employer_name or
  employer_location) is non-null across the mentions that map to it.

MENTION MAPPING RULES — CRITICAL:
- Every mention in the input MUST appear in mention_mappings exactly once.
  Do not skip or duplicate any mention.
- Use only the mention IDs shown in the input. Do not invent IDs.
- Every entity_id in mention_mappings must reference an entity defined in
  the entities array.
- entity_ids must be unique sequential integers starting from 1.

The document is provided as a separate attachment. It contains all employer
mentions for one person, each labeled with a mention ID.
