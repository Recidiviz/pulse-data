Field Definitions:

* release_location_type: Classify the proposed release location into exactly
  one of three types:
  - 'rented_or_owned_residence': The person plans to live in a private
    residence they rent or own. Use this when the note describes an address
    check or home plan for a private home or apartment, even if no street
    address is given. Also use this as the default when the note mentions a
    release address without indicating a facility or a dependent host.
  - 'named_facility': The proposed location is a specific named shelter,
    halfway house, transitional housing program, treatment center, sober living
    house, or other named organization (e.g., "Oxford House", "Fort Collins
    Rescue Mission", "Community Corrections Center"). Use this when a proper
    facility name is mentioned.
  - 'dependent': The person plans to live with a family member, friend, or
    partner who will host them. Use this when the note indicates the release
    plan involves staying with or at the home of another person (e.g., "plans
    to reside with mother", "will stay with friend John", "approved to live
    with partner").

* proposed_release_address: The full proposed physical address where the person
  plans to reside after release. Include all available components: street
  address, apartment/unit number, city, state, and zip code. Standardize
  formatting when possible (title case for street names, consistent
  abbreviations for St, Ave, Blvd, Dr, etc.).
  - Populate for any release_location_type when a street address is available —
    this may be the person's own address, a named facility's address, or the
    address of the dependent host's home.
  - Only extract addresses explicitly tied to release or reentry planning. Do
    not extract facility addresses, mailing addresses, or addresses mentioned
    in passing that are not clearly identified as the proposed release location.
  - If multiple proposed addresses are mentioned (e.g., a primary and a
    backup), extract only the primary or most recent proposed address.
  - If the note references an address investigation but does not include the
    actual address text, use null_reason='no_info_found'.

* proposed_release_location_name: The proper name of the named facility
  proposed as the person's release location (e.g., "Fort Collins Rescue
  Mission", "Oxford House", "Denver Rescue Mission", "Community Corrections
  Center").
  - Only populate when release_location_type is 'named_facility'.
  - Only use a proper name — do not populate with generic descriptions like
    "shelter" or "halfway house".
  - Both proposed_release_location_name and proposed_release_address may be
    non-null when release_location_type is 'named_facility' and the note also
    provides the facility's street address.

* proposed_dependent_housing_relationship_type: The relationship of the person
  to the individual(s) they plan to live with upon release.
  - Only populate when release_location_type is 'dependent'.
  - Always use the most specific value available:
    - 'father' / 'mother': Use when the parent's gender is clear.
    - 'parent_unspecified': Use when a parent is mentioned but gender is not
      specified (e.g., "plans to live with a parent"). Only use if you cannot
      determine father or mother.
    - 'sister' / 'brother': Use when the sibling's gender is clear.
    - 'sibling_unspecified': Use when a sibling is mentioned but gender is not
      specified (e.g., "will reside with sibling"). Only use if you cannot
      determine sister or brother.
    - 'cousin': Use when the host is described as a cousin.
    - 'relatives_other': Use when the note refers to an unspecified family
      member or multiple household members (e.g., "plans to live with family",
      "approved to live with mother and siblings", "will stay with a family
      member", "mother and a friend"). Only use if none of the more specific
      values apply.
    - 'friend': Use when the host is described as a friend (and there is only
      one host).
    - 'partner': Use for a romantic partner, spouse, or significant other.
    - 'other': Use when the host relationship is described in a way that does
      not fit any of the above values.

* investigation_status: Determine ONE value from: 'pending_investigation',
  'investigation_in_progress', 'approved', 'denied', 'conditionally_approved'.
  This status applies to whichever location type is being investigated.
  - 'pending_investigation': A location has been proposed but the
    investigation or verification process has not yet started (e.g.,
    "address submitted for review," "awaiting home check," "pending
    verification," "application submitted").
  - 'investigation_in_progress': The investigation or verification of the
    proposed location is actively underway (e.g., "home visit scheduled,"
    "field check in progress," "investigation ongoing," "address being
    verified," "application awaiting response").
  - 'approved': The proposed location has been approved, confirmed, or
    cleared for the person to reside at upon release (e.g., "address
    approved by parole," "home plan confirmed," "residence verified and
    accepted," "community placement approved").
  - 'denied': The proposed location has been denied, rejected, or found
    unsuitable (e.g., "address denied by parole," "home plan rejected,"
    "residence not approved," "community placement denied").
  - 'conditionally_approved': The proposed location has been tentatively
    approved with conditions or contingencies that must be met (e.g.,
    "approved pending background check of household member," "approved
    contingent on removal of firearms," "conditionally accepted").

Clarifications:
1. Only extract locations explicitly tied to release or reentry planning.
2. If the note discusses a release plan but the location has been changed or

3. If a note mentions that an investigation was completed but does not

   investigation_status rather than guessing the result.
4. A location that was previously denied but has a new investigation in

5. Do not conflate the status of different locations. If location A was
   denied and location B is pending, only extract location B with
